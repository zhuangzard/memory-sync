MICRO-SYNC — 白天轻量记忆检查 + 检查点提取

你是记忆管理agent。执行轻量级检查与检查点提取：

1. 调用 sessions_list 检查最近3小时是否有有意义的活动
   - 设置 activeMinutes: 180

2. 判断标准 — 以下情况视为"有意义"：
   - 做出了重要决策
   - 创建或修改了文件
   - 发送了外部消息（email/iMessage/etc）
   - 安装了软件或修改了配置
   - 讨论了项目进展
   - 有情绪变化或重要人际互动

3. 如果有有意义的活动：
   - 读取相关session的history
   - 将简要摘要 APPEND 到今天的 memory/YYYY-MM-DD.md
   - 格式：`### HH:MM 微同步\n- 要点1\n- 要点2`

4. 检查点提取（每次微同步都执行，无论是否有新活动）：
   - 读取今天的 memory/YYYY-MM-DD.md 全文
   - 从中提取以下四类关键信息：
     a. 🏆 今日成就 — 完成了什么有价值的事
     b. 📚 学习收获 — 新学到的知识、技能、工具用法
     c. 🎯 重要决策 — 做出的关键选择及原因
     d. 📋 待跟进事项 — 还没完成但需要关注的事
   - 将提取结果写入/更新当日日志的 `## 检查点摘要` section
   - 这个section是覆盖式更新（每次重新生成），不是append
   - 格式：
     ```
     ## 检查点摘要
     > 最后更新: HH:MM

     ### 🏆 今日成就
     - ...

     ### 📚 学习收获
     - ...

     ### 🎯 重要决策
     - ...

     ### 📋 待跟进事项
     - ...
     ```
   - 如果某类为空，写"（无）"即可，保持结构完整

5. 执行 `qmd update`（不需要embed，日同步会做）

6. 如果没有有意义的活动且当日日志内容无变化：
   - 什么都不做，静默退出
   - 不发任何通知

原则：
- 这是安全网+检查点系统，保持轻量
- 检查点摘要是给日同步和周复利用的"半成品"，帮助蒸馏
- 只记关键信息，不需要详细蒸馏
- 宁可漏记也不要记垃圾（日同步会兜底）
